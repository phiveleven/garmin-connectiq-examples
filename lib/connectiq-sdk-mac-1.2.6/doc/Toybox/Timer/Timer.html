Overview

A Timer class will call a function after a number of milliseconds. There are two types of timers: one-shot or repeating. A one-shot timer will only run once while a repeating timer will call the function every X milliseconds until stop() is called. If a repeating timer fails to run before its next execution time then any missed executions will be skipped.

Instance Method Details

- (Object) start(callback, time, repeat)

Use start() to start the timer. The given callback will be called no sooner than the given number of milliseconds. This will cause an error if too many timers are set. The number of available timers (default of 3) and the minimum time value (default of 50ms) depends on the host system.

Parameters:

  • callback (Method) —

    A function to call after the timer completes

  • time (Number) —

    The number of milliseconds to wait before invoking callback

  • repeat (Boolean) —

    Set to true to have the timer repeat until stop() is called

Since:

  • 1.0.0

- (Object) stop

Use stop() to stop the timer from running. This only needs to be called for repeating timers. The Timer can be started again by calling start().
