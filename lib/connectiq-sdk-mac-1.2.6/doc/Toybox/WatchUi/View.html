Overview

View is the base class for a drawable page. It handles the lifecycle of each View in an app. For widgets and watch-apps the lifecyle is onStart(), onLayout(), onShow(), onUpdate() and onHide(). For watchfaces and datafields the lifecycle is onLayout(), onShow() and onUpdate(). For datafields, if the size changed since the last onUpdate(), onLayout() will be called prior to onUpdate(). SimpleDataField objects don't have the same life cycle. onLayout(), onShow(), and onUpdate() aren't called for SimpleDataField objects.

- (Boolean) onUpdate(dc)

When a View is active, onUpdate() is used to update dynamic content. This function is called when the View is brought to the foreground. For widgets and watch-apps it is also called when WatchUi.requestUpdate() is called. For watchfaces it is called once a minute and for datafields it is called once a second. If a class that extends View does not implement this function then any Drawables contained in the View will automatically be drawn.
